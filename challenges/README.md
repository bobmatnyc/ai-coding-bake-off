# Challenges

Five Python challenges of increasing complexity. Each challenge directory contains:

- `PROBLEM.md` --- Full problem description (what agents read)
- `test_suite/` --- Provided test cases and fixtures (agents must pass these)
- `evaluation/rubric.md` --- Scoring rubric for evaluation

## Challenge Overview

| Level | Name | Time Budget | Key Skills Tested |
|-------|------|-------------|-------------------|
| 1 | [Markdown Table Formatter](level-1-table-formatter/) | ~30 min | CLI, file I/O, string formatting |
| 2 | [Git Log Analyzer](level-2-git-analyzer/) | ~1 hr | Parsing, project structure, packaging |
| 3 | [Weather Alerting Service](level-3-weather-alerter/) | ~2 hr | REST API, SQLite, scheduling, Docker |
| 4 | [Document Processing Pipeline](level-4-doc-pipeline/) | ~3-4 hr | Architecture, extensibility, NLP, search |
| 5 | [Team Task Board](level-5-task-board/) | ~6-8 hr | Full-stack, WebSocket, auth, Docker Compose |

## Rules

- Challenges are **read-only** during the competition phase.
- Agents may read PROBLEM.md and run the test suite but must not modify them.
- Agents should write additional tests beyond the provided suite.
- Each agent works independently and must not view other agents' solutions.
